<title id="_65f08cf0-017d-119b-ebe9-cc583c78d9ed">Present</title>
<p id="_1f68de3b-8430-7468-e8a9-06db8b40fb9b">In order to test for this operation of the <tt>CG_Discovery</tt> interface, some parameters have to be fixed in the request, these are: <tt>Iteratorsize(100)</tt>, <tt>Cursor(0)</tt>, <tt>Presentation(full)</tt> and <tt>ResultSetID(ResultSetID sent by the query)</tt>. Other parameters and their possible values are shown in <xref target="fig3"/>.</p>

<p id="_d5f4ed3c-11c6-349e-9cd8-ad39d28e92e1">The general approach which applies to the testing of the capabilities of present is equal to the general approach of the query method (see <xref target="query"/>). First of all the Catalog Certification Program will set the <tt>Present</tt> capability to not supported but in case one of the calls was successful the capability will be considered supported. If a successful status is retrieved, the last check will be using the NV format, i.e., the program will check if the functionality of this method is working testing the results retrieved with NV format. If the NV format is working the present capability will be shown as supported if the number of hits retrieved is the number of hits expected by the test.</p>
